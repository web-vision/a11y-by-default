# Documentation

## What should be documented

* What does it do
* Installation
* Usage
* Configuration

Screen https://docs.typo3.org/p/web-vision/deepltranslate-core/main/en-us/Index.html as good example.
 
The normal TYPO3 extension documentation has sections for developers, integrators and editors. The documentation
must be well-structured and give context to the cases, the extension is used.

COnfiguration and backend have to be pictured with screenshots.

A technical guideline is not necessarily needed. A description of the functionality is needed.

## Screenshots

Take screenshots for the relevant parts of the PageLayout addition and the module itself.

Take screenshots from the backend user and group fields.

For taking screenshots, use the provided TYPO3 credentials from the AGENTS.md.

Generate a normal editor with full pagetree access and access to the accessibility module. Use the switch user
functionality for taking editor relevant screenshots.

Focus on the functionality in the screenshots but ensure the context around is given. Full-size backend screenshots
should be avoided.

## How to document

Document in a precise, semi-technical language. The documentation must be readable by editors, integrators and developers.

Follow the [TYPO3 Documentation guidelines](https://docs.typo3.org/m/typo3/docs-how-to-document/main/en-us/Advanced/CodingGuidelines.html)
and use the [rst cheat sheet](https://docs.typo3.org/m/typo3/docs-how-to-document/main/en-us/Basics/RstCheatSheet.html)

## Where to document

The extension contains a Documentation directory holding rst files.
